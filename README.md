# 💰 SpendingAnalyser — 跨平台个人消费分析工具

支持**支付宝、微信、京东、美团**四大平台的交易数据导入、自动分类、退款对冲、交互式可视化分析。

## ✨ 主要功能

| 功能 | 说明 |
|---|---|
| **四平台解析** | 自动识别并解析支付宝、微信、京东、美团的 CSV 账单 |
| **退款对冲** | 智能匹配退款与原始消费，计算净支出 |
| **双轨分类** | 自动区分「消费」和「资金流动」（转账、理财等） |
| **LLM 自动打标** | 使用大模型对消费记录进行 L1（一级）+ L2（二级）分类 |
| **标签继承** | 手动修改过的标签在重新处理时自动保留 |
| **交互式仪表盘** | 支持分类饼图、趋势图、商户排行、多维度筛选、下钻分析 |
| **在线编辑** | 可直接在仪表盘中修改分类标签 |

## 🚀 快速开始

### 环境要求

- **Python 3.9+**（Mac 自带或 `brew install python3`）
- **Node.js 18+**（`brew install node`）

### 1. 准备数据

将你的账单 CSV 文件放入 `data/` 目录：

| 平台 | 文件命名示例 | 获取方式 |
|---|---|---|
| 支付宝 | `支付宝交易明细(20250101-20251231).csv` | 支付宝 App → 账单 → 导出 |
| 微信 | `微信支付账单(20250101-20250331).csv` | 微信 → 钱包 → 账单 → 导出 |
| 京东 | `京东交易流水*.csv` | 京东 App → 我的 → 交易流水 → 导出 |
| 美团 | `美团账单(20250101-20250331).csv` | 美团 App → 我的 → 账单 → 导出 |

> 每个平台可以放多个文件（比如按季度导出的微信账单），系统会自动合并去重。

### 2. 配置 LLM（可选）

如果需要 LLM 自动分类功能，编辑 `config.env`：

```env
LLM_API_KEY=你的API密钥
LLM_BASE_URL=https://api.xiaomimimo.com/anthropic
LLM_MODEL=mimo-v2.5-pro
```

> 不配置 LLM 也可以使用，只是新增的交易记录需要手动在仪表盘中编辑分类。

### 3. 一键启动

```bash
chmod +x start.sh
./start.sh
```

首次运行会自动安装依赖（约 1-2 分钟），之后每次启动约 3 秒。
启动完成后浏览器会自动打开 **http://localhost:3001**。

### 4. 停止服务

```bash
./start.sh stop
```

## 🤖 LLM 打标

首次启动后，系统会生成待打标的 batch 文件。运行以下命令进行自动分类：

```bash
python3 src/classifiers/llm_tagger_runner.py
```

打标完成后重启服务即可在仪表盘看到分类结果：

```bash
./start.sh stop && ./start.sh
```

## 🎯 仪表盘使用指南

### 筛选功能
- **多选筛选**：平台、一级分类、二级分类均支持同时选择多个
- **日期范围**：支持自定义起止日期
- **排除分类**：排除不想看到的类别（如「爱车养车」）

### 交互式下钻
- **饼图**：点击任意扇区 → 进入二级分类视图
- **趋势图**：点击柱状条 → 自动筛选到该时间段
- **分类排行**：点击排行项 → 按该分类筛选
- **商户排行**：点击商户 → 跳转到交易明细并搜索
- **下钻状态栏**：每次下钻后顶部显示蓝色状态条，可单独退出或全部退出

### 排序功能
- 点击交易明细表的任意列头 → 按该列升序/降序排序

### 编辑分类
- 在交易明细表中点击「编辑」→ 修改 L1/L2 分类 → 保存
- 修改会自动保存到 `tag_overrides.csv`，下次重新处理时自动继承

## 📊 数据处理流程

```
原始 CSV → 解析 → 退款对冲 → 消费/资金分类 → 标签继承 → LLM 打标 → 仪表盘展示
```

1. **解析**：自动识别4个平台的 CSV 格式，统一为标准字段
2. **退款对冲**：将退款金额从对应消费中扣除，退款行标记忽略
3. **双轨分类**：区分消费（日常花费）和资金流动（转账、理财、充值等）
4. **标签继承**：上次手动修改过的分类自动沿用
5. **LLM 打标**：未分类的消费记录发送给大模型自动归类

## ❓ 常见问题

### Q: 如何更换 LLM 服务商？
编辑 `config.env`，修改 `LLM_BASE_URL` 和 `LLM_MODEL`。只要兼容 Anthropic API 格式即可。

### Q: 打标不满意怎么办？
在仪表盘中手动编辑分类，修改会被记住。如需重新打标某些 batch，删除 `output/tagging_batches/batch_XXX_result.json` 后重新运行打标命令。

### Q: 如何给其他用户使用？
1. 拷贝整个项目文件夹
2. 让用户把自己的账单放入 `data/` 目录
3. 编辑 `config.env` 填写 API Key
4. 运行 `./start.sh`

### Q: 端口被占用怎么办？
运行 `./start.sh stop` 清理旧进程后重新启动。
